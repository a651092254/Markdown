
# 一·标题的写法
在想要设置标题的文字前面加#来表示  
一个#是一级标题，两个#是二级标题，以此类推总共分6级
语法如下：
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```
效果如下：
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题


# 二·字体的写法

一个星号包起来是斜体  
两个星号包起来是加粗  
三个星号就是加粗的斜体  
两个波浪号包起来是删除  
示例：
```
*斜体字体*  
**加粗字体**  
***斜体又加粗字体***  
~~删除线字体~~
```

*斜体字体*  
**加粗字体**  
***斜体又加粗字体***  
~~删除线字体~~


# 三·样式的写法
三个以上的星号就是分割线  
大于号就是引用，可以写多个大于号就是多级嵌套  
换行要在上一行末尾加1个以上的空格

```
***
---
>一级引用
>>二级引用
>>>三级引用

第一行文字（不包含空格）
第二行文字


第一行文字（包含空格的）  
第二行文字
```

***
---
>一级引用
>>二级引用
>>
>>>三级引用

第一行文字（不包含空格）
第二行文字


第一行文字（包含空格的）  
第二行文字

# 七·列表的写法
### 1.无序列表是用 - + * 中的一种来修饰即可
语法如下：
```
+ 这个是加号修饰的
- 这个是减号修饰的
* 这个是星号修饰的
注意：符号和内容之间要留有空格才会生效
```
效果如下：
+ 这个是加号修饰的
- 这个是减号修饰的
* 这个是星号修饰的

### 2.有序列表是用数字和一个点来修饰即可
语法如下：
```
1. 数字标记有序列表
2. 数字标记有序列表
3. 数字标记有序列表

a. 尝试小写字母标记
b. 尝试小写字母标记

A. 尝试大写字母标记
B. 尝试大写字母标记

当然最后证明只有数字可以生效
```
效果如下：
1. 数字标记有序列表
2. 数字标记有序列表
3. 数字标记有序列表

a. 尝试小写字母标记
b. 尝试小写字母标记

A. 尝试大写字母标记
B. 尝试大写字母标记
### 3.列表之间是支持相互嵌套的，上级和下级之间敲三个空格即可,更简单的可以敲一下tab键代替
语法如下：

  + 这是第二层的无序列表
  + 这是第二层的无序列表

  + 二层无序列表

  2. 第二层的有序列表

   * 这是第二层的无序列表

# 八·表格的写法
**单元格之间用|来分割，第二行分割表头和表单，内容用“-”来填充，填一个就可以起作用，图美观可以写多个来对齐；
文字默认是靠左，如果需要靠右就在分割行的填充符号右侧加“：”，居中就在两侧加。
每行头上两侧的竖线可加可不加，效果相同，加上规范，不加就是图方便。**
语法如下：
```

|表头..1|表头..2|表头..3|
|------|-----:|:----:|
|值1|值2|值3|
值4|值5|值6
```
效果如下：

|表头..1|表头..2|表头..3|
|--|--:|:--:|
|值1|值2|值3|
值4|值5|值6

# 九·进阶技能
## 1.流程图
首先定义小模块，统一格式为 “模块名称 =>模块类型:模块内容”；  
之后定义流程，统一格式为“模块名称（方向，条件）->下一模块名称”  

模块类型：  
start/end：圆形框  
operation:矩形流程框
condition:菱形条件框  

    ```flow
    st=>start: 开始
    op1=>operation: 流程框
    c=>condition: 判断框
    e=>end: 结束
    
    st(right)->op1(right)->c
    c(yes)->e
    c(no)->op1
    ```

```flow
st=>start: 开始
op1=>operation: 流程框
c=>condition: 判断框
e=>end: 结束
st(right)->op1(right)->c
c(yes)->e
c(no)->op1
```
## 待办事项表
类似于无序列表，前面用-[ ]来修饰，[ ]前后都要加空格，未办的中间要用空格填充。
示例：
``` 
- [ ] 未办事项
- [ ] 未办事项
- [x] 已办事项
- [x] 已办事项
- [x] 已办事项
```
示例效果：
- [ ] 未办事项
- [ ] 未办事项
- [x] 已办事项
- [x] 已办事项
- [x] 已办事项
